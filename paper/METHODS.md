# Methods draft — ego-mol-llm

*Manuscript-ready methods text (v0.1). Adapt journal style as needed.*

## Overview

We developed **ego-mol-llm**, an open-source workflow that predicts the molecular structure of an unknown MS/MS feature from its **ego neighborhood** in a spectral molecular network. The method does not require raw fragment peak lists at inference time; instead it uses network topology and library annotations of neighboring nodes (precursor *m/z*, cosine similarity, absolute mass differences, optional SMILES) as evidence for a chemistry large language model (LLM).

## Molecular network input

Networks are provided as GraphML exports consistent with GNPS / HNSW-style molecular networking. Nodes carry precursor mass (`PEPMASS`), optional library name and SMILES, community labels, and flags indicating direct spectral neighbors of a query. Edges store pairwise cosine similarity of MS/MS spectra and absolute precursor mass differences.

## Blind ego-context construction

For a query (seed) node, we extract the top *N* adjacent nodes ranked by cosine similarity (default *N* = 25) and optionally a set of two-hop annotated nodes to provide broader chemical context. In **blind evaluation mode**, the seed’s library name and SMILES are withheld from the model. This mirrors a realistic “unknown” annotation scenario while enabling quantitative comparison against held-out labels.

## Chemistry LLM backends

Structure hypotheses are generated by open LLMs via a chat interface:

1. **ChemDFM-v2.0-14B** — chemistry foundation model obtained by domain pre-training and instruction tuning of **Qwen2.5-14B** (Zhao et al., *Cell Reports Physical Science*, 2025; OpenDFM release).
2. **ChemDFM-R-14B** — reasoning-enhanced chemical LLM in the same family.
3. **ChemDFM-v1.5-8B** — smaller chemistry model suitable for consumer GPUs.
4. Optional general instruct models (**Qwen2.5**, **Qwen3.5**) for ablation studies.

Inference is supported via Hugging Face Transformers (optional 4-bit quantization) or any OpenAI-compatible HTTP server (e.g. Ollama, vLLM).

## Prompting

The model receives a system prompt specifying the annotation task, evidence hierarchy (high cosine + explicit structures preferred; weak isobars discounted), and a required JSON schema with fields: `smiles`, `iupac_or_common_name`, `formula`, `adduct`, `confidence`, `rationale`, and `alternatives`. User content enumerates the query precursor *m/z* and ordered neighbor evidence.

## Post-processing

Model text is parsed for the JSON object. SMILES strings are validated and canonicalized with RDKit when available. Predicted neutral mass is compared to the observed precursor *m/z* under common adduct hypotheses ([M−H]⁻, [M+H]⁺, [M+Na]⁺) with a configurable absolute mass tolerance (default 0.05 Da). Runs export the full prompt, raw completion, structured prediction, and an ego-network figure for auditability.

## Evaluation protocol (recommended)

1. Collect GraphML ego networks with trusted ground-truth structures for seed nodes.
2. Run blind predictions across model backends with fixed temperature and seeds where applicable.
3. Score: valid SMILES rate; exact structure match (InChIKey first block); Tanimoto similarity of Morgan fingerprints; formula match; mass consistency.
4. Report calibration of model-stated confidence vs empirical accuracy.
5. Compare against non-LLM baselines (most similar neighbor’s structure; majority scaffold among top-*k* cosine neighbors).

## Limitations

Ego-network LLM annotation inherits library annotation errors and spectral isobar confusion (e.g. peptides near small-molecule *m/z*). It is complementary to de novo spectral structure elucidation (SIRIUS, MSNovelist, DiffMS, etc.), not a replacement. Model hallucinations remain possible; expert review is required.
